## 🔴 Critical Issues (Phải sửa ngay)

### 1. Lộ lọt thông tin nhạy cảm (Plain-text API Keys)

- **Vị trí:** `config.json` (File này được sinh ra ở thư mục gốc sau khi chạy app).
- **Nguy hiểm:** API key của Google Gemini được lưu ở dạng văn bản thuần túy (plain text). Nếu dự án này được đưa lên GitHub hoặc chia sẻ cho người khác, bất kỳ ai cũng có thể lấy và sử dụng API key của bạn, dẫn đến việc bị tính phí hoặc khóa tài khoản.
- **Cách sửa:**
  - Chuyển sang sử dụng biến môi trường (`.env`) để lưu API key.
  - Mã hóa (encrypt) giá trị key trước khi lưu vào JSON nếu muốn giữ cách lưu cũ.
  - Quan trọng nhất: Cần tạo file `.gitignore` để chặn việc upload `config.json` lên mạng.

## 🟡 Warnings (Nên sửa)

### 1. Thiếu file `.gitignore`

- **Triệu chứng:** Thư mục `.venv`, `.pytest_cache`, `__pycache__` và `config.json` đang nằm phơi bày.
- **Hậu quả:** Nếu anh dùng Git, anh sẽ vô tình đẩy hàng GB dữ liệu rác và thông tin mật lên máy chủ.
- **Phác đồ:** Tạo ngay file `.gitignore` tiêu chuẩn cho Python/PySide6.

### 2. Mâu thuẫn trong quản lý thư viện (Dependency Inconsistency)

- **Triệu chứng:** File `requirements.txt` yêu cầu `google-genai`, nhưng code hiện tại đang chặn thư viện này do lỗi treo máy.
- **Hậu quả:** Người dùng mới cài đặt sẽ mất công tải thư viện nặng nhưng app lại không dùng đến, gây bối rối khi bảo trì.
- **Phác đồ:** Đồng bộ lại `requirements.txt` hoặc sửa triệt để lỗi treo của `google-genai`.

### 3. Phụ thuộc vào thư viện cũ (Deprecated SDK)

- **Triệu chứng:** App đang dùng `google-generativeai` (SDK cũ).
- **Hậu quả:** Google đã thông báo ngừng hỗ trợ bản này. Trong tương lai gần, app có thể ngừng hoạt động đột ngột nếu Google tắt server bản cũ.
- **Phác đồ:** Lên kế hoạch chuyển dịch sang `google-genai` (SDK mới) một khi giải quyết được lỗi treo môi trường.

### 4. Hiệu năng tạo ảnh (Serial Image Generation)

- **Triệu chứng:** Việc tạo ảnh diễn ra tuần tự từng slide một.
- **Hậu quả:** Nếu tạo 20 slide, người dùng phải đợi rất lâu.
- **Phác đồ:** Cân nhắc sử dụng `QThreadPool` để tạo ảnh song song cho nhiều slide cùng lúc.

## 🟢 Suggestions (Tùy chọn)

### 1. "Và rò rỉ" encoding trong `main.py`

- **Ghi chú:** Có đoạn code ép kiểu `sys.stdout` sang `utf-8`. Đây là cách sửa nhanh nhưng không phải là "best practice" lâu dài. Nên xử lý triệt để qua biến môi trường `PYTHONIOENCODING`.

### 2. Cấu trúc Database

- **Ghi chú:** Hiện tại cột `outline_json` lưu cả cục dữ liệu lớn. Nếu sau này có tính năng tìm kiếm theo nội dung slide, việc này sẽ làm app chạy chậm đi một chút.

## ⚠️ NEXT STEPS (Phác đồ điều trị)

1. **Khẩn cấp:** Tạo file `.gitignore` và tách API key ra khỏi file config chính.
2. **Ổn định:** Cập nhật `requirements.txt` đúng thực tế.
3. **Nâng cao:** Tối ưu hóa hiệu năng tạo ảnh song song.
